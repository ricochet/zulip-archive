Topic: Improving CLI Args/Config Names Consistency

Robin Brown (Sep 19 2023 at 18:13):

It would be nice for users if the same concept (e.g. world name, wit path) used the same command line arguments / config file parameter names across different guest languages. Right now Componentize-Py and Componentize-JS currently use different names (wit-path vs. wit, world-name vs. world) and use -w to mean two different things. I haven't done a survey of Cargo Component's options but they should ideally match too. What do we think would be a sensible default choice for how we name these things that we can apply to all the tools?
