This update brings in a number of features such as:

  • The component model binary format and AST has been slightly adjusted in a few locations. Names are dropped from parameters/results now in the internal representation since they were not used anyway. At this time the ability to bind a multi-return function has not been exposed.

  • The wasmparser validator pass will now share allocations with prior functions, providing what's probably a very minor speedup for Wasmtime itself.

  • The text format for many component-related tests now requires named parameters.

  • Some new relaxed-simd instructions are updated to be ignored.

I hope to have a follow-up to expose the multi-return ability to the embedding API of components.
