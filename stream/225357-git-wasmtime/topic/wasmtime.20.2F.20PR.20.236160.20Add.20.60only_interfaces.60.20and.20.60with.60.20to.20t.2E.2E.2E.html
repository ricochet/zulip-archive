Add the only_interfaces and with options to the bindgen! macro. These two
additional options enable the definitions of a use of bindgen! to be split up
and reused in the following way:

  • Passing only_interfaces: true to an invocation of bindgen! will cause only
    the interfaces depended on by the supplied world to be generated, allowing the
    interface definitions depended on by two world files to be shared.

  • Passing with: { "a": module::name } allows references to the a import to
    be remapped to a known module, in this case module::name. When this option
    is supplied, no code will be generated for the interface a, assuming that
    those definitions came from a previous use of bindgen! with
    only_interfaces: true.
