Hi @theoparis -- as you can see here

error: Wasmtime is being compiled for an architecture that it does not support

Wasmtime doesn't (yet?) support riscv32 (of any variant). The implementation of a Cranelift backend will be the major hurdle here: as discussed in a few places, this is a pretty large (several month) project. We're happy to keep around tracking issues for currently unsupported ISAs, and I can't find one for riscv32 at the moment, so if you'd like to open one for that, feel free! However given the lack of support at this time this build error is not a bug: it's correctly reporting the lack of support.
