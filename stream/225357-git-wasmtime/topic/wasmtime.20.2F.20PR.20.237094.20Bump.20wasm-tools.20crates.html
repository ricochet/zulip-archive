Two major changes/reasons for this update:

  • Primarily pulling in support for semicolons-in-WIT files. Semicolons are not currently required, though, so I'll follow-up later with actual semicolons.

  • The syntax for parsing (if ...) was fixed in wast. Previously it did not require (then ...) but this is required by the spec. New spec tests require this as well. This breaks existing text format tests which don't use (then ...) inside of an (if ...). Most tests were updated by hand but embenchen_* tests were updated by running through the old parser to produce non-s-expression using code.
